# OPLINK CLI

The OPLINK CLI provides commands to manage your AI-driven workflows in a standardized manner.

## Basic Commands

- **`oplink init`**
  Initializes OPLINK in a project. Creates a `.mcp-workflows` directory and a `oplink.config.mjs` file if needed.
  ```bash
  npx oplink init
  ```
  Options:
  - `--dir <directory>`: Target directory to initialize
  - `--ide <cursor|windsurf|cline|rootcode|headless>`: Choose an IDE integration or run headless
  - `--force`: Overwrite existing config files

- **`oplink add <preset>`**
  Adds a built-in preset or a preset from a giget source to `.mcp-workflows`.
  ```bash
  npx oplink add coding
  ```
  or
  ```bash
  npx oplink add github:user/repo/path
  ```
  Options:
  - `--force`: Overwrite existing preset file/directory
  - `--cwd`: Custom working directory

- **`oplink config get`**
  Displays the resolved OPLINK configuration from `oplink.config.mjs`.

## Examples

1. **Initialize in `my-project`**:
   ```bash
   npx oplink init --dir my-project
   ```
2. **Add a built-in preset**:
   ```bash
   npx oplink add coding
   ```
3. **Add from a giget source**:
   ```bash
   npx oplink add github:username/presets/coding
   ```
